# Decisions

## D-001 원본 보존 경계

- 결정: 기존 `ExportedProject/`, `AuxiliaryFiles/`를 원본 보존 영역으로 취급한다.
- 이유: 이미 16.75 GB의 산출물이 존재하므로 같은 워크스페이스 안에서 즉시 전체 복제하면 공간만 중복된다.
- 보완: 전체 SHA-256 manifest를 생성했다. 외부 물리 백업 위치는 사용자 저장 장치 지정 후 확정한다.

## D-002 새 프로젝트 위치

- 결정: 새 프로젝트는 `UnityProject/`에 둔다.
- 이유: 원본 산출물과 import/cache/settings를 물리적으로 분리하기 위해서다.

## D-003 Unity 버전

- 결정: 새 프로젝트 메타데이터의 목표는 `2022.3.62f3`로 둔다.
- 근거: 마스터 가이드 우선 버전이며 ExportedProject에도 해당 버전이 기록되어 있다.
- 검증: 로컬 Unity `2022.3.62f3`에서 package 해석, compile, Scene 저장·재로드를 확인했다. 2022.3.2f1은 비교 환경으로 유지한다.

## D-004 이번 작업 범위

- 결정: Phase 0 정적 조사를 완료하고 Scene 단위 복원으로 전환한다.
- 제외: 사용자 요청 없는 빌드, 커밋, 푸시 및 검증되지 않은 전체 에셋 일괄 import.

## D-005 코드 출처

- 결정: ExportedProject의 C#은 provenance가 밝혀질 때까지 `Recovered 후보`로 표시한다.
- 이유: 원본 소스와 DLL 역추출 산출물을 현재 증거만으로 구분할 수 없다.
