# 원본 보존 및 Manifest

## 상태

- 조사 중

## 복원 수준 목표

- R1

## 목표

- `ExportedProject/`와 `AuxiliaryFiles/`를 불변 기준 자료로 확정한다.
- 경로, 크기, SHA-256을 포함한 재생성 가능한 manifest를 만든다.
- 저장 공간을 확인한 뒤 별도 물리 백업 필요성을 결정한다.

## 현재 근거

- 원본 후보 Assets는 17,539개 파일, 약 16.75 GB다.
- 현재 워크스페이스는 작업 시작 시 Git 저장소가 아니었다.
- 원격 GitHub 저장소는 ref가 없는 빈 저장소다.

## 변경 금지

- 원본 파일 이동, 이름 변경, 수정, 삭제 금지
- manifest를 원본 폴더 안에 생성하지 않음

## 완료 조건

- [ ] 원본 전체 파일 목록 생성
- [ ] 모든 파일의 SHA-256 생성
- [ ] `.meta`, ProjectVersion, manifest 존재와 해시 기록
- [ ] 원본/작업 프로젝트 경계 문서화
- [ ] 별도 백업 위치와 용량 결정

## 검증

- manifest 행 수와 실제 파일 수 일치
- 무작위 표본 해시 재계산 일치
- 생성 전후 원본 파일 수/크기/mtime 변화 없음

## 남은 불확실성

- 16 GB 이상 전체 해싱에 필요한 시간
- 별도 백업 드라이브 위치와 가용 용량
